# 건설업 홈페이지 — 히어로 섹션 블록 로딩 애니메이션 개발 요청서

---

## 1. 개요

건설업 홈페이지 메인 페이지(히어로 섹션)에 진입 시, 정사각형 블록(픽셀)들이 점진적으로 나타나면서 최종 화면을 드러내는 인트로 애니메이션을 구현합니다. 레퍼런스는 CAMER STUDIO 웹사이트의 픽셀 모자이크 마스크 효과이며, 블록 크기를 약 20배 확대하여 건설업의 "벽돌/블록/건축" 이미지와 연결합니다.

---

## 2. 레퍼런스 분석

### CAMER STUDIO 원본 특징
- 작은 정사각형 픽셀들이 이미지 위에 모자이크 마스크로 겹쳐짐
- 블록이 가장자리에서 불규칙하게 분포하며, 이미지를 부분적으로 가리거나 드러냄
- 흰색 블록 + 어두운 이미지의 대비
- 블록들이 이미지 경계 바깥으로도 튀어나가면서 디지털 글리치/해체 느낌

### 우리 버전 차별점
- 블록 크기를 원본 대비 약 20배 확대 (약 80~120px 정사각형)
- 건설업과 연결되는 "건축 블록/벽돌이 쌓여 올라가는" 내러티브
- 블록이 쌓이면서 화면을 가리고 있다가 → 하나씩 사라지면서 히어로 콘텐츠가 드러나는 방식
- 다크모드 아님 — 밝고 세련된 톤

---

## 3. 애니메이션 상세 시나리오

### Phase 1 — 초기 상태 (0ms)
- 화면 전체가 정사각형 블록으로 빈틈없이 덮여 있음 (그리드 형태)
- 블록 컬러: 브랜드 컬러 계열 그라데이션 (아래 컬러 섹션 참고)
- 히어로 콘텐츠(텍스트, 이미지)는 블록 뒤에 이미 로드되어 있지만 보이지 않음

### Phase 2 — 블록 해체 시작 (0ms ~ 800ms)
- 화면 중앙에서부터 블록이 사라지기 시작
- 사라지는 방식: 각 블록이 개별적으로 `opacity: 1 → 0` + `scale(1) → scale(0.8)` + 약간의 `translateY(-10px)` (위로 살짝 떠오르듯 사라짐)
- 중앙에서 바깥으로 방사형(radial)으로 퍼져나가며 사라짐
- 각 블록의 사라지는 타이밍에 랜덤 지터(±50~100ms) 추가 → 기계적이지 않고 자연스러운 느낌
- 완전히 규칙적이면 안 됨. 약간의 불규칙성이 핵심

### Phase 3 — 콘텐츠 노출 (400ms ~ 1200ms)
- 블록이 사라진 자리로 히어로 콘텐츠가 보이기 시작
- 모든 블록이 사라질 필요 없음 — 가장자리 일부 블록(10~15개)은 잔존
- 잔존 블록은 히어로 이미지/콘텐츠 위에 장식적으로 겹쳐져 레퍼런스 이미지처럼 "디지털 해체" 무드 연출
- 잔존 블록 위치: 주로 화면 모서리와 가장자리, 콘텐츠 핵심 영역(텍스트, CTA)은 반드시 비워야 함

### Phase 4 — 최종 상태 (1200ms~)
- 히어로 텍스트: 별도의 `fadeIn + translateY(20px → 0)` 애니메이션으로 등장 (Phase 3 시작 후 200ms 딜레이)
- CTA 버튼: 텍스트 등장 후 추가 200ms 딜레이로 등장
- 잔존 블록들: 매우 느린 `float` 애니메이션 (위아래 3~5px, 4~6초 주기) → 정적이지 않고 살아있는 느낌
- 스크롤 시: 잔존 블록들이 패럴랙스로 콘텐츠보다 느리게 움직임 (선택사항)

### 전체 타임라인 요약

| 시점 | 이벤트 | 지속시간 |
|------|--------|---------|
| 0ms | 블록 그리드가 화면 전체를 덮고 있음 | — |
| 0~200ms | 짧은 정적 상태 (블록이 화면을 채운 채 대기) | 200ms |
| 200~1000ms | 블록들이 중앙→바깥으로 사라짐 | 800ms |
| 600~1200ms | 히어로 텍스트 fadeIn | 600ms |
| 800~1400ms | CTA 버튼 fadeIn | 600ms |
| 1200ms~ | 잔존 블록 float 애니메이션 루프 | 무한 |

## 4. 블록 상세 스펙

### 크기
- 기본: **100px × 100px** (뷰포트 1920px 기준)
- 반응형 비율 유지: `calc(100vw / 19)` 정도로, 가로 약 19개 블록이 채워지는 크기
- 모바일(< 768px): **60px × 60px**으로 축소
- 태블릿(768~1024px): **80px × 80px**

### 블록 간 간격
- gap 없음 (빈틈없이 채움) — 또는 1~2px gap으로 미세한 그리드 라인 표현 (선택)

### 블록 컬러
건설업 브랜드에 맞는 컬러 팔레트. 모든 블록이 같은 색이 아니라, 아래 팔레트 중 랜덤 배정:

```
Primary:    #1A1A1A  (차콜 — 콘크리트/철골)     배정 비율 30%
Secondary:  #D4C5A9  (샌드 베이지 — 모래/벽돌)   배정 비율 25%
Tertiary:   #8B8680  (웜 그레이 — 시멘트)        배정 비율 20%
Accent:     #E8A63A  (건설 옐로 — 중장비/안전)   배정 비율 10%
Light:      #F5F0E8  (오프화이트 — 밝은 벽)      배정 비율 15%
```

각 블록에 위 비율에 따라 랜덤으로 색상 배정. 인접 블록끼리 같은 색이 반복되지 않도록 체크 로직 추가.

### 블록 정렬
- CSS Grid 또는 absolute position 기반
- 화면 전체를 빈틈없이 채우되, 뷰포트 바깥으로 넘치는 블록은 `overflow: hidden` 처리
- 블록 그리드는 뷰포트 크기에 맞춰 동적으로 행/열 수 계산

## 5. 이징(Easing) & 모션 커브

### 블록 사라지는 애니메이션
```css
transition: opacity 400ms cubic-bezier(0.4, 0, 0.2, 1),
            transform 400ms cubic-bezier(0.4, 0, 0.2, 1);
```
- `cubic-bezier(0.4, 0, 0.2, 1)` = Material Design "Standard" 이징
- 부드럽게 시작해서 자연스럽게 끝남

### 텍스트/CTA 등장
```css
transition: opacity 600ms cubic-bezier(0, 0, 0.2, 1),
            transform 600ms cubic-bezier(0, 0, 0.2, 1);
```
- `cubic-bezier(0, 0, 0.2, 1)` = "Decelerate" 이징
- 빠르게 나타나서 부드럽게 안착

### 잔존 블록 Float
```css
animation: float 5s ease-in-out infinite alternate;

@keyframes float {
  0% { transform: translateY(0px); }
  100% { transform: translateY(-5px); }
}
```

## 8. 성능 요구사항

- 애니메이션은 **GPU 가속** 속성만 사용 (`transform`, `opacity`). `width`, `height`, `top`, `left` 변경 금지
- 블록 수가 많으므로(약 190개), DOM 노드 최적화 필요. `will-change: transform, opacity` 적용하되 애니메이션 완료 후 제거
- 애니메이션 완료 후 사라진 블록은 DOM에서 제거하거나 `display: none` 처리 → 렌더 트리 부하 최소화
- `requestAnimationFrame` 기반 타이밍 또는 CSS animation-delay 활용
- Lighthouse Performance 점수 90+ 유지. 인트로 애니메이션이 LCP(Largest Contentful Paint)를 지연시키지 않도록, 히어로 콘텐츠는 블록 뒤에서 즉시 렌더되어야 함
- 사용자의 `prefers-reduced-motion: reduce` 설정 시, 애니메이션 스킵하고 즉시 최종 상태 표시

## 9. 기술 구현 가이드

### 추천 방식: CSS Grid + JS 타이밍 제어

```
블록 생성: JS로 뷰포트 크기 계산 → 필요한 행/열 수 산출 → div 생성
블록 배치: CSS Grid (repeat(auto-fill, 100px))
사라지는 순서: JS로 각 블록의 중심 좌표 → 화면 중앙까지 거리 계산 → 거리 기반 delay 배정
애니메이션: CSS transition + JS로 클래스 토글
```

### 대안: Canvas / WebGL
- 블록 수가 300개 이상으로 늘어날 경우 Canvas 기반이 성능상 유리
- 단, 접근성(스크린 리더 등)과 SEO 측면에서 CSS/DOM 방식이 권장됨

### 프레임워크별 참고
- **React/Next.js**: `useEffect`에서 IntersectionObserver 또는 마운트 시 애니메이션 트리거. Framer Motion의 `stagger` 활용 가능
- **Vue/Nuxt**: `onMounted` + GSAP `stagger` 또는 Vue Transition Group
- **Vanilla JS**: `requestAnimationFrame` + CSS custom properties로 delay 관리

## 10. 접근성 & 폴백

- `prefers-reduced-motion: reduce` 감지 시 → 애니메이션 전부 스킵, 히어로 콘텐츠 즉시 표시
- 인트로 영역에 `aria-hidden="true"` 적용 (장식 요소)
- 히어로 텍스트/CTA에는 적절한 시맨틱 태그 (`h1`, `a`, `button`)
- 저사양 기기 감지(navigator.hardwareConcurrency < 4 등) 시 블록 수 축소 또는 애니메이션 간소화

## 11. 재방문 처리

- 첫 방문: 전체 인트로 애니메이션 재생
- 같은 세션 내 재방문: 애니메이션 스킵 (sessionStorage 플래그 활용)
- 새 세션: 다시 재생
- 선택사항: 스킵 버튼 제공 ("Skip Intro" 우하단 작게 표시, 탭/클릭 시 즉시 최종 상태로)
